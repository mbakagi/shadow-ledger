# AI Assistant Behavioral Guidelines

The following rules apply to all interactions within this project:

## 1. Token Efficiency & Conciseness
- Be highly concise. Do not use redundant reasoning or unnecessary words.
- Provide straight-to-the-point answers and direct solutions.
- Minimize conversational filler (e.g., "I have fixed the issue!", "Here is what I found:").
- Omit long-winded explanations unless specifically asked for.

## 2. Code Modifications & Output
- Do not repeat or print out entire files just to show a small change.
- When explaining code changes, highlight ONLY the changed lines or the specific logic altered.
- Avoid restating the user's prompt or requirements back to them.

## 3. General Workflow
- Execute tasks efficiently with the minimum required steps.
- Prioritize action (editing files, running commands) over verbose planning or explaining what you are *about* to do, unless a complex architectural plan requires approval.
